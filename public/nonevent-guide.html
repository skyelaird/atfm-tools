Non-event CTOT portal — usage guide

A self-serve slot system for pilots flying across CTP event airspace without a booked event slot. Open for testing / validation now through post-CTP.

For pilots

What this is

A self-serve slot system for pilots flying across CTP event airspace on a CTP event day without a booked event slot. It issues you a calculated take-off time (CTOT) that threads you through the event traffic without hogging capacity the booked flights need.

When to use it

You're flying, and all of these are true:

  • There's an active CTP event (check ctp.vatsim.net)
  • You're NOT booked into the event
  • Your departure OR arrival crosses / is near NAT corridors on event day (12-22Z on 2026-04-25)

If any of those don't apply, you don't need a CTOT from us.

What you need before requesting

  • Callsign you'll be logging in with
  • ADEP: 4-letter origin ICAO
  • ADES: 4-letter destination ICAO
  • EOBT: intended off-block time in Zulu (ISO 8601, e.g. 2026-04-25T14:00:00Z)
  • Aircraft type: ICAO code (e.g. B738, A320, B77W)
  • Route: filed route string, SIDs/STARs stripped (enroute portion only)
  • Cruise altitude: filed FL (e.g. 370)

Steps

  1. Sign in with VATSIM Connect (button at top of /nonevent.html)
  2. Check ECFMP first — open ecfmp.vatsim.net/dashboard in another tab. Note any flow measures that apply to your route (mandatory reroutes, level caps). If you file a route that violates an active measure, our system will reject your request.
  3. Fill in the form → Submit
  4. The server either:
    • Issues a CTOT — hold your block until the CTOT. You have a 15-min window; if you haven't taken off by CTOT + 15, your slot auto-releases.
    • Rejects with one of:
      • CTP_EXCLUDED: ADEP or ADES is booked-traffic-only. Pick another airport.
      • ECFMP_REJECT: an active ECFMP measure blocks your route. The message includes the measure's ident, reason, and (for mandatory reroutes) the required routing text. Refile and try again.
      • CANADIAN_HANDLED_ELSEWHERE: ADEP or ADES is one of the 7 Canadian event airports metered by the main atfm-tools allocator. File and go.
  5. Wait at your CTOT — squawk standby, don't take TSAT earlier than CTOT-2 min. Taking off before CTOT can trigger follow-up ATC intervention.
  6. If you miss your CTOT — submit a new request. The new request goes to the next free slot at ADES (usually within 15 min).
Stable CTOT: the time is frozen once issued. Subsequent wind-forecast updates won't shift it.

For ATCOs

What's different for you

If you're signed in with an ATCO-rated VATSIM Connect account (S1 / S2 / S3 / C1 / C3 / I1 / I3 / SUP / ADM), you get:

  • Live dashboard of every active non-event CTOT across all airports
  • Flight-plan puller — enter a pilot's callsign and the system pulls their filed plan from our ingestor cache and auto-fills the request form. Click Submit on their behalf.
  • Ability to cancel any slot (e.g. pilot abandoned the flight)

Typical workflow

  1. Pilot calls you on text/voice asking for a CTOT (common on VATSIM-UK / VATCAN / VATUSA fields without a local CD)
  2. Open /nonevent.html, sign in, switch to the ATCO tab
  3. Type the pilot's callsign in the flight-plan puller
  4. System fills ADEP/ADES/EOBT/route/type/FL from our ingestor
  5. Confirm with the pilot, Submit
  6. Read the returned CTOT back to the pilot

When to cancel a slot

  • Pilot disconnected and isn't coming back
  • Pilot refiled a different route and we issued a fresh slot for it (the old one is stale)

Reference: ECFMP flow-measure types

If you get an ECFMP_REJECT, the measure.type field will be one of:

TypeWhat it means
MANDATORY_ROUTEYou MUST file the route shown in measure.value. Refile and retry.
PROHIBITYour filed route is prohibited during the measure window. File an alternate.
GROUND_STOPAll departures matching the filter are grounded. Wait until the measure expires.
MILES_IN_TRAIL, PER_HOUR, MINIMUM_DEPARTURE_INTERVALRate / spacing restriction. Our allocator honours it automatically; CTOT is shifted to respect the rate.
MAX_IAS, MAX_MACH, IAS_REDUCTION, MACH_REDUCTIONSpeed restriction en route. Not blocked at issuance; honour it in flight.

See ecfmp.vatsim.net/dashboard for the full active list.

FAQ

Why are Canadian airports blocked?

The main atfm-tools allocator already issues CTOTs at CYHZ / CYOW / CYUL / CYVR / CYWG / CYYC / CYYZ based on AAR. Non-event flights to/from those airports go through the main system, not this one.

What's the difference between a CTP event CTOT and a non-event CTOT?

Event CTOTs are published weeks in advance through the CTP booking system. Non-event CTOTs are issued on-demand by this portal for pilots flying alongside the event.

Why 4 slots per hour?

Light metering. 4/hr = 15-minute intervals, sustainable for any airport without starving normal traffic. Reasonable default; can be tuned per-airport later.

Where's the CDM plugin output?

Not in this layer. Most target users (US pilots) don't run the CDM plugin. The portal is the delivery channel.

What if I'm in Canadian airspace but my ADEP/ADES isn't a CTP / Canadian airport?

You still get a CTOT from this system. The Canadian-airport block only fires if your ADEP OR ADES is one of the 7 Canadian airports we manage directly.

What if the ECFMP API is down?

We cache the last known measure set. If the cache is also empty (cold start + outage), we fail open — no ECFMP checks happen and your request goes through based on the rest of the rules. A banner on the portal shows "ECFMP unavailable".

Can I see someone else's CTOT?

Pilots see only their own. ATCOs see the full active list.
