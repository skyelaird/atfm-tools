Reports — User Guide

This page shows prediction accuracy and traffic statistics for the 7 Canadian airports managed by atfm-tools. Data refreshes automatically every 60 seconds. Use the time-window pills (6h / 12h / 24h / 48h / 7d) to change the reporting period.

Stat cards

The top row summarises the selected window:

  • Movements — completed arrivals (with ALDT) + departures (with ATOT).
  • ELDT err — median prediction error. How far our landing estimate was from the actual landing, in minutes. Lower is better; target ≤3 min.
  • Dwell — median time from pilot spawn to pushback. Indicates how long pilots sit at the gate before moving.
  • TLDT err — median slot fidelity. How close the frozen landing slot matched reality.

Values are medians (not means) to resist outliers. Numbers dimmed when sample size < 5.

ETA source distribution

Shows which estimation method was used for each flight’s final ELDT. Higher-priority sources produce better accuracy:

SourceDescription

FILEDPilot’s filed enroute time (usually from SimBrief, wind-corrected)
OBSERVED_POSGeometric: distance from current position ÷ speed

CALC_DEFAULTDistance ÷ 430 kt (last resort, lowest confidence)

TLDT flights table

Individual flight records — columns ordered to tell the story left-to-right: who the flight is → departed → naive baseline vs actual → our prediction vs actual → source tier.

  • Flt / ETE — observed flight time / filed ETE (e.g. 188/570). Large discrepancy suggests mid-connect.
  • ATOT+ETE — naive baseline prediction: actual takeoff + filed enroute time.
  • ALDT — actual landing. Placed between the two predictions so both errors can be read visually.
  • ETA err — (ATOT+ETE) − ALDT. How wrong the naive baseline was.
  • TLDT / TLDT err — our frozen prediction at T-90m and its error vs ALDT. Compare with ETA err to see the gain from our model.
  • PARTIAL / SIM Nx badges — outliers. PARTIAL = pilot connected mid-flight; SIM = accelerated sim rate. Both dim and excluded from accuracy stats.
  • Error colour: green ≤3m, yellow 3–5m, red >5m.

Aircraft type table

Counts of completed movements by ICAO type code. Per-airport columns show arrivals + departures at that airport. Total is the sum across all airports.

Data window auto-refreshes every 60 s. ATOT = Actual Take-Off (first seen airborne). AOBT = Actual Off-Block (first pushback detected). ALDT = Actual Landing. AIBT = Actual In-Block. AXOT = Actual Taxi-Out (ATOT minus AOBT). AXIT = Actual Taxi-In (AIBT minus ALDT). ELDT = Estimated Landing (system prediction, frozen at T-90m). TLDT = Target Landing (allocated slot time). CTOT = Calculated Take-Off (slot issued by allocator). MAE = Mean Absolute Error. Per EUROCONTROL Airport CDM Manual v5.0. Dwell = median spawn-to-pushback (AOBT minus first_seen), capped 120 min. Error buckets (≤3 / 3–5 / 5–10 / >10) are mutually exclusive, sum to 100%. Dimmed when n < 5. Partial-observation flights (mid-connect, observed < 50% of filed ETE) excluded from accuracy stats.
